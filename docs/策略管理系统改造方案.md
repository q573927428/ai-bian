# 策略管理系统改造方案

> 币安永续合约 AI 自动交易系统 - 策略自由配置系统

---

## 一、现状分析

### ✅ 现有优势（可复用）
1. **指标计算体系完善**：EMA/RSI/MACD/ATR/OI 等指标计算逻辑已存在
2. **指标缓存系统成熟**：TTL + LRU + 请求去重 + 降级机制
3. **交易执行模块完整**：开仓/平仓/监控/止损止盈逻辑健全
4. **风控系统完善**：熔断/冷却/每日限制/移动止损
5. **AI 集成已实现**：DeepSeek API 调用 + AI 自学习系统
6. **状态持久化**：JSON 文件存储 + 重启恢复

### ❌ 现有痛点（需改造）
1. **策略配置写死在 JSON**：无法前端动态创建/编辑策略
2. **单策略模式运行**：不支持多策略并发执行
3. **指标获取分散**：不同策略重复获取相同数据
4. **交易逻辑硬编码**：`analyzer.ts` 中信号生成逻辑固定
5. **版本控制缺失**：策略更新无版本管理，无法回滚
6. **仓位管理简单**：未实现"同交易对单仓位"限制

## 三、数据模型设计

### 3.1 策略定义 (`types/strategy.ts`)

```typescript
// 策略定义
interface Strategy {
  id: string;                    // 策略唯一ID (UUID)
  name: string;                  // 策略名称
  description: string;           // 策略描述
  version: number;               // 版本号 (从1开始递增)
  createdAt: string;             // 创建时间
  updatedAt: string;             // 更新时间
  isActive: boolean;             // 是否激活
  
  // 市场数据配置
  marketData: MarketDataConfig;
  
  // 技术指标配置
  indicators: IndicatorConfig[];
  
  // 统计数据配置
  statistics: StatisticsConfig[];
  
  // AI 提示词配置
  aiPrompt: AIPromptConfig;
  
  // 风险管理配置
  riskManagement: RiskConfig;
  
  // 执行配置
  executionConfig: ExecutionConfig;
  
  // 版本历史
  versionHistory: VersionHistory[];
}

// 市场数据配置
interface MarketDataConfig {
  symbols: string[];             // 交易对列表 ["BTC/USDT", "ETH/USDT"]
  timeframes: string[];          // K线周期 ["15m", "1h", "4h", "1d"]
  klineLimit: number;            // K线获取数量 (默认300)
}

// 技术指标配置
interface IndicatorConfig {
  type: 'EMA' | 'RSI' | 'MACD' | 'ATR';
  params: Record<string, number>; // 参数，如 { period: 14 }
  timeframes: string[];          // 适用的时间周期
  enabled: boolean;              // 是否启用
}

// 统计数据配置
interface StatisticsConfig {
  type: 'OI' | 'Volume';         // OI持仓量 / 成交量
  params: Record<string, number>; // 参数
  timeframes: string[];          // 适用的时间周期
  enabled: boolean;
}

// AI 提示词配置
interface AIPromptConfig {
  systemPrompt: string;          // 系统提示词（固定部分）
  userPrompt: string;            // 用户提示词（交易逻辑）
  temperature: number;           // AI 温度 (0-1)
  maxTokens: number;             // 最大token数
  model: string;                 // AI模型 (deepseek-chat)
}

// 风险管理配置
interface RiskConfig {
  maxRiskPercentage: number;     // 单笔最大风险 (%)
  stopLossATRMultiplier: number; // 止损ATR倍数
  takeProfitRatios: number[];    // 止盈盈亏比 [2.5, 3.5]
  maxDailyTrades: number;        // 每日最大交易次数
  maxDailyLoss: number;          // 每日最大亏损 (%)
  trailingStop: TrailingStopConfig;
}

// 移动止损配置
interface TrailingStopConfig {
  enabled: boolean;
  activationRatio: number;       // 激活盈亏比
  trailDistance: number;         // 跟踪距离 (ATR倍数)
  minMoveDistance: number;       // 最小移动距离 (%)
}

// 执行配置
interface ExecutionConfig {
  scanInterval: number;          // 扫描间隔 (秒)
  leverage: number | 'dynamic';  // 杠杆 (固定值或动态)
  marginMode: 'cross' | 'isolated'; // 保证金模式
  positionMode: 'one-way' | 'hedge'; // 持仓模式
}

// 版本历史
interface VersionHistory {
  version: number;
  updatedAt: string;
  changes: string;               // 变更说明
  snapshot: Strategy;            // 该版本的完整配置快照
}
```

```typescript
interface TradeSignal {
  strategyId: string;            // 触发策略ID
  symbol: string;                // 交易对
  direction: 'long' | 'short';   // 方向
  action: 'open' | 'close' | 'hold'; // 动作
  price: number;                 // 当前价格
  stopLoss: number;              // 止损价
  takeProfit?: number[];         // 止盈价数组
  confidence: number;            // 置信度 (0-100)
  leverage?: number;             // 建议杠杆
  positionSize?: number;         // 建议仓位大小
  reasoning: string;             // AI 决策理由
  indicators: IndicatorValues;   // 触发时的指标值
  timestamp: string;             // 信号时间
}

// 指标值（AI返回的结构化数据）
interface IndicatorValues {
  ema?: { fast: number; slow: number };

  macd?: { macd: number; signal: number; histogram: number };
  atr?: number;
  oi?: { value: number; trend: string };
  volume?: { current: number; average: number };
}
```

```typescript
   * 取消订阅
   */
  unsubscribe(strategyId: string): void;

   * 获取指标数据（自动去重+缓存）
   */
  async getIndicators(symbol: string, timeframe: string, types: string[]): Promise<IndicatorData>;
  
  /**
   * 预加载数据（在策略启动时批量加载）
   */

  /**
   * 定时更新（按时间周期自动刷新）
   */
  private startUpdateLoop(): void;

```

**关键特性：**
1. **智能合并**：多个策略请求相同交易对+周期时，只获取一次数据
2. **按需更新**：根据 K 线周期自动设置 TTL（15m 线 15 分钟过期）
3. **降级策略**：API 失败时返回缓存，缓存失效时返回 null
4. **事件驱动**：数据更新后自动通知所有订阅策略

```typescript
private async analyzeSymbol(strategyId: string, symbol: string) {
  const strategy = this.getStrategy(strategyId);
  
  // 1. 从指标中心获取数据（自动复用缓存）
  const indicators = await this.indicatorsHub.getIndicators(
    symbol, 
    strategy.marketData.timeframes,
    strategy.indicators.map(i => i.type)
  );
  
  // 2. 构建 AI 提示词
  const prompt = this.buildPrompt(strategy, indicators);
  
  // 3. 调用 AI 分析（带缓存，相同数据10分钟内不重复调用）
  const signal = await this.callAI(strategyId, symbol, indicators);
  
  // 4. 验证信号
  if (!this.validateSignal(signal)) return;
  
  // 5. 检查仓位限制（同交易对互斥）
  if (await this.positionManager.hasConflict(symbol, strategyId)) {
    logger.warn(`策略 ${strategyId} 的信号被忽略，交易对 ${symbol} 已有仓位`);
    return;
  }
  
  // 6. 执行交易
  await this.executeSignal(signal);
}
```

### 4.5 AI 分析器改造

```typescript
// server/utils/ai-analysis.ts (改造后)

class AIAnalyzer {
  /**
   * 通用 AI 分析方法
   */
  async analyze(
    strategyId: string, 
    symbol: string, 
    promptConfig: AIPromptConfig,
    indicators: IndicatorData,
    marketData: MarketData
  ): Promise<TradeSignal> {
    // 1. 检查缓存
    const cacheKey = this.buildCacheKey(strategyId, symbol, indicators);
    const cached = await this.cache.get(cacheKey);
    if (cached) return cached;
    
    // 2. 构建完整提示词
    const fullPrompt = this.buildFullPrompt(promptConfig, indicators, marketData);
    
    // 3. 调用 AI API
    const response = await this.callDeepSeek(fullPrompt, promptConfig);
    
    // 4. 解析 AI 返回的 JSON
    const signal = this.parseResponse(response);
    
    // 5. 验证信号格式
    this.validateSignal(signal);
    
    // 6. 缓存结果
    await this.cache.set(cacheKey, signal, 600000); // 10分钟
    
    return signal;
  }
  
  /**
   * 构建提示词模板
   */
  private buildFullPrompt(
    promptConfig: AIPromptConfig,
    indicators: IndicatorData,
    marketData: MarketData
  ): string {
    return `
${promptConfig.systemPrompt}

## 当前市场数据
交易对: ${marketData.symbol}
价格: ${marketData.price}
时间: ${new Date().toISOString()}

## 技术指标
${this.formatIndicators(indicators)}

## 交易逻辑
${promptConfig.userPrompt}

请返回 JSON 格式的交易信号。
`;
  }
}
```

## 八、关键问题解决方案

### Q1: 如何保证指标数据统一获取且高效复用？

**方案：**
- `IndicatorsHub` 作为唯一数据入口
- 内部维护请求队列，相同 `symbol + timeframe + indicator` 只请求一次
- 按 K 线周期设置 TTL（15m 线 15 分钟，1h 线 1 小时）
- 使用发布-订阅模式，数据更新后自动推送给所有订阅策略

### Q2: 如何实现"同一交易对只能持有一个仓位"？

**方案：**
- `PositionManager` 维护 `symbol -> strategyId` 映射表
- 开仓前调用 `canOpenPosition()` 检查
- 开仓成功后 `lockSymbol()` 锁定
- 平仓后 `unlockSymbol()` 释放
- 程序启动时从交易所同步实际持仓重建映射表

### Q3: 如何支持策略版本更新且不影响运行中的实例？

**方案：**
- 每次更新创建新版本（version++）
- 保存旧版本快照到 `versionHistory`
- 运行中的策略实例继续使用旧配置
- 用户手动点击"重启策略"后加载新配置
- 支持一键回滚到任意历史版本

### Q4: AI 调用成本如何优化？

**方案：**
- 智能缓存：相同指标数据 10 分钟内不重复调用
- 条件触发：仅当指标发生显著变化时才调用 AI
- 批量分析：多个交易对合并为一次 AI 请求
- 降级策略：API 限流时使用缓存结果

### Q5: 如何复用现有代码？

**方案：**
- ✅ **完全复用**：`indicators-core.ts`、`indicators-entry.ts`、`position-opener.ts`、`position-closer.ts`、`position-monitor.ts`、`trailing-stop.ts`
- ✅ **改造复用**：`ai-analysis.ts`（增加多策略支持）、`indicators-cache.ts`（升级为 IndicatorsHub）
- ❌ **重写**：`analyzer.ts`（改为 AI 驱动）、`scanner.ts`（改为 StrategyEngine）
